### 2. Effective Prompt Templates

#### For Coding Tasks
```
Act as a senior [language] developer. 
Review this code for [specific concern].
Provide:
1. Issues found
2. Fixed code
3. Explanation of changes

Code: [paste code]
```

#### For Learning/Understanding
```
Explain [concept] like I'm a [skill level] developer.
Include:
- Core principles
- Common pitfalls
- Practical examples
- Resources for deeper learning
```

#### For Problem Solving
```
I'm facing this problem: [describe]
Constraints: [list]
Goal: [state desired outcome]

Think through this step-by-step and suggest solutions.
```

#### For Code Review
```
Review this code as a senior engineer:
- Security issues
- Performance bottlenecks
- Code quality/style
- Suggested improvements

[Paste code]
```

## Advanced Techniques

### 1. Chain-of-Thought Prompting
```
Let's think through this step-by-step:
1. First, understand the problem
2. Break it into components
3. Solve each component
4. Verify the solution

Problem: [your problem]
```

### 2. Role-Playing
```
You are an expert [role] with [X years] experience.
Your task is to [specific task].
Approach this with [specific mindset/approach].
```

### 3. Iterative Refinement
```
First pass: [initial request]
Second pass: Now make it more [specific quality]
Third pass: Optimize for [different quality]
```

### 4. Multi-Modal Reasoning
```
Analyze this from multiple angles:
1. Technical perspective
2. Business perspective
3. User experience perspective
4. Security perspective

Topic: [your topic]
```

## Grok-Specific Tips

### Leverage Grok's Strengths
- **Rebellious streak**: Ask for unconventional solutions
- **Real-time knowledge**: Query current events (if enabled)
- **Longer context**: Provide detailed background
- **Code generation**: Specify language, framework, style

### Known Limitations
- May not have latest libraries (cutoff date)
- Can't execute code (text-only)
- May need clarification on ambiguous requests

```
Generate a Python script that:

- Creates issues automatically
- Uses the gh CLI

Include error handling and logging.
```

### 2. Documentation
```
Write README documentation for:
- Project: [name]
- Audience: [developers/users]
- Key features: [list]

```

### 3. Debugging Help
```
I'm getting this error: [paste error]
My code: [paste relevant code]
Expected behavior: [describe]
Actual behavior: [describe]

What's wrong and how do I fix it?
```

```
Provide:
```

```
Create a 30-day learning plan for:

Include daily tasks, resources, and projects.
```

## Common Pitfalls & How to Avoid Them

### âŒ Too Vague
**Bad:** "Help me with code"
**Good:** "Debug this Python function that's returning None instead of the expected list"

### âŒ Missing Context
**Bad:** "Why isn't this working?"
**Good:** "I'm trying to [goal]. I tried [approach]. Getting [error]. Expected [result]."

### âŒ No Constraints
**Bad:** "Build me a website"
**Good:** "Build a landing page for [product] using [framework] with [specific features]"

### âŒ Single Massive Prompt
**Bad:** 2000-word prompt asking for everything
**Good:** Break into conversational steps
